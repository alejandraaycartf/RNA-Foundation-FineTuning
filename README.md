# RNA-Foundation-FineTuning

Official repository for the Bachelor's Thesis: **"Design of fine-tuning pipelines in transcriptomic foundation models for the analysis of omics datasets"**.

This project explores the adaptation of deep learning architectures applied to transcriptomics, comparing the predictive performance and biological knowledge extraction of classical Machine Learning models against the transcriptomic foundation model **GeneRAIN**. Furthermore, it implements parameter-efficient fine-tuning using **LoRA** and utilizes Tabular Variational Autoencoders (**TVAE**) to mitigate data scarcity through the generation of synthetic samples.

---

## Project Objectives

1. **Evaluate predictive performance:** Compare classical approaches (LASSO, Random Forest, MLP) on raw data against the latent projections (200-dimensional embeddings) generated by GeneRAIN, both in its frozen version (zero-shot) and after adaptation with LoRA.
2. **Biological interpretability:** Determine whether the foundation model's attention mechanisms favor the identification of genetic signatures with greater functional richness and biological specificity, quantified through the Mean Information Content (MIC) and Functional Abundance (FA).
3. **Data augmentation:** Evaluate the impact of incorporating synthetic transcriptomic data (generated with TVAE) to mitigate noise and sample scarcity in complex clinical cohorts, such as Parkinson's Disease (PPMI).

---

## Workflow (Pipeline)

The code is structured into a sequential experimentation flow where the output of one stage feeds into the next:

*   **1. Data Download:** The starting point consists of downloading the raw transcriptomic count files from public repositories such as **GTEx**, ARCHS4, and PPMI.
*   **2. Preprocessing and Gene Selection (`split.py`, `DEG.py`):** These scripts strictly take the raw count matrices as input. `split.py` performs data partitioning (80/10/10) and extracts highly variable genes (HVG). Meanwhile, `DEG.py` takes the raw counts and clinical labels to identify differentially expressed genes using PyDESeq2.
*   **3. Data Augmentation (`TVAE.py`):** Based on the previous training partition and the filtered gene lists, it synthesizes transcriptomic profiles using a Tabular Variational Autoencoder to balance and augment the available data.
*   **4. Classical Evaluation (`baseline.py`):** This script takes the combined data generated by `split.py`, `DEG.py`, and `TVAE.py`. It optimizes and evaluates the baseline models (LASSO, Random Forest, MLP) on the standardized data.
*   **5. Latent Extraction - Phase 1 (`phase1.py`):** Similar to the baseline, this script takes the previously processed data from `split.py`, `DEG.py`, and `TVAE.py`, along with the original pre-trained weights of GeneRAIN. It applies the *Binning-By-Gene* normalization, extracts the static *embeddings* from the frozen model (zero-shot), and trains an MLP classifier.
*   **6. Fine-Tuning - Phase 2 (`phase2.py`):** To run, this script requires the data, features, and base model generated in Phase 1. It performs the fine-tuning of the architecture by injecting LoRA adapters into the *Transformer's* attention matrices.
*   **7. Biological Interpretability (`run_enrichment.py`, `sweep_topk.py`, `plot_sweep.py`):**
    *   `run_enrichment.py` takes the gene rankings (LASSO coefficients, RF importance, and GeneRAIN's attention matrices) along with the *background* file (gene universe of the problem). It connects to the g:Profiler API and returns the static MIC and FA metrics.
    *   `sweep_topk.py` takes the results from the phases and ranking metrics to automate the calculation by performing a dynamic sweep of the Top-K variable, generating a global CSV file.
    *   `plot_sweep.py` takes the previous global CSV to generate the final multiple-sensitivity line plots in `.png` format.

## Case Studies

The project has been empirically validated across three scenarios with varying levels of predictive difficulty:

1. **Trivial Difficulty (Positive Control):** Tissue differentiation between brain frontal cortex and blood (GTEx/ARCHS4 data).
2. **Intermediate Difficulty:** Morphological classification of tissues from the same organ (sigmoid colon vs. transverse colon).
3. **High Difficulty:** Detection of idiopathic Parkinson's Disease in peripheral blood samples, using the Parkinson's Progression Markers Initiative (PPMI) clinical cohort.

## Requirements, Installation, and Directory Structure

To ensure computational reproducibility, running this project in a Conda environment using a GPU is recommended. Main dependencies include `python 3.13.12`, the `pytorch` ecosystem with CUDA, `calm-data-generator` (for synthetic data generation), `pydeseq2`, `optuna`, `scanpy`, `anndata`, `umap-learn`, and `gprofiler-official`.

**Strict integration with GeneRAIN:**
This project is designed to run **inside the root directory of the official GeneRAIN repository**. To set up the environment correctly, follow these steps in order:

### 1. Clone the repositories and prepare the environment
Clone the original GeneRAIN repository and copy the `RNA-Foundation-FineTuning` scripts inside it (for example, into the `src/` folder).

```bash
git clone https://github.com/suzheng/GeneRAIN.git
cd GeneRAIN
# Copy the RNA-Foundation-FineTuning scripts here
```

### 2. Patch GeneRAIN's native normalization script
Since our custom pipeline already feeds pre-processed and normalized data into the foundation model, we must prevent GeneRAIN from applying a redundant library-size scaling and log10 transformation. 

Open the file `src/data/GetBinsByGeneForNewSamples.py` from the GeneRAIN repository and modify the `scale_by_sample_total_count_log_transform` function to bypass the transformation. Comment out the scaling lines so it looks exactly like this:

```python
def scale_by_sample_total_count_log_transform(gene_by_sample_expr_mat, output_prefix, min_total_count=1000000, min_expr_genes=0):
    expression_data = gene_by_sample_expr_mat
    total_counts = np.sum(expression_data, axis=0)
    non_zero_counts = np.count_nonzero(expression_data, axis=0)
    mean_counts = np.mean(expression_data, axis=0)
    median_counts = np.median(expression_data, axis=0)
    max_counts = np.max(expression_data, axis=0)
    valid_samples = (total_counts >= min_total_count) & (non_zero_counts > min_expr_genes)
    expression_data = expression_data[:, valid_samples]
    
    # --- COMMENTED OUT TO AVOID DOUBLE NORMALIZATION ---
    # total_counts_valid = total_counts[valid_samples]
    # scaling_factors = 10000000 / total_counts_valid
    # normalized_expression = expression_data * scaling_factors[np.newaxis, :]
    # log_transformed_expression = np.log10(normalized_expression + 1)
    
    log_transformed_expression = expression_data
    # ---------------------------------------------------

    stats_df = pd.DataFrame({
        'total_read_counts': total_counts,
        'non_zero_genes': non_zero_counts,
        'mean_read_counts': mean_counts,
        'median_read_counts': median_counts,
        'max_read_counts': max_counts,
        'valid_samples': valid_samples
    })
    
    if output_prefix != None:
        stats_df.to_csv(f'{output_prefix}_stats.tsv', sep='\t', header=True, index=False)
    return log_transformed_expression.astype(np.float32), valid_samples, stats_df
```

### 3. Download GeneRAIN resources
You must download the files provided by the original GeneRAIN authors (available on their Zenodo/attachments) and place them **exactly** in the following paths:

*   **Pretrained model weights (`.pth`):**
    Place it in: `data/models/GeneRAIN.BERT_Pred_Genes_Binning_By_Gene.pth`.
*   **Configuration files (`.json`):**
    Place them in: `jsons/exp3_BERT_Pred_Genes_Binning_By_Gene.param_config.json`.
*   **ARCHS4 base data (for Binning):**
    You must download the `.npy` and `.txt` files from ARCHS4 and place them in: `data/external/ARCHS/normalize_each_gene/`. 
    *(Example: `human_gene_v2.2_with_zero_expr_genes_bin_tot2000_gene2vec_0.005_subsampled.npy` and its corresponding `.txt` symbol file)*.


### 4. Create the project directory structure
Our Fine-Tuning and Data Augmentation pipeline uses a dynamic directory structure based on the `PROBLEM_NAME` variable. The scripts assume that raw data, partitions, gene lists, and TVAE synthetic data are strictly separated by problem.

Run the following command in your terminal (while at the root of `GeneRAIN`) to instantly create all the necessary folders for the 3 study cases (`ppmi`, `colon_sigmoid_vs_colon_transverse`, and `frontal_cortex_vs_blood`):

```bash
# Create the directory structure for data and results
for PROBLEM in "ppmi" "colon_sigmoid_vs_colon_transverse" "frontal_cortex_vs_blood"; do
    # Folders for data preprocessing
    mkdir -p data/processed/$PROBLEM/splits
    mkdir -p data/processed/$PROBLEM/genes
    
    # Independent folders for TVAE synthetic data (HVG and DEG)
    mkdir -p data/processed/$PROBLEM/synthetic/combined
    mkdir -p data/processed/$PROBLEM/synthetic/combined_deg
    
    # Folders to save results and models for each phase
    mkdir -p results/$PROBLEM/baseline
    mkdir -p results/$PROBLEM/phase1
    mkdir -p results/$PROBLEM/phase2
done

# Create directories for raw external data
mkdir -p data/external/GTEX
mkdir -p data/external/PPMI
```

### 5. Location of input data (Raw Data)
Before running the partitioning scripts, you must place the raw data downloaded from GTEx or PPMI into their corresponding folders:
*   GTEx `.gct.gz` files must go into `data/external/GTEX/`.
*   PPMI clinical and omic files must go into `data/external/PPMI/`.

Once this structure is ready, the pipeline (running in order: `split.py`, `DEG.py`, `TVAE.py`, `baseline.py`, `phase1.py`, and `phase2.py`) will automatically feed from these folders, avoiding any path-related errors.

## Usage (Execution)

Given the strict data dependency flow, scripts must be executed in order:
1. `split.py`
2. `DEG.py`
3. `TVAE.py`
4. `baseline.py`
5. `phase1.py`
6. `phase2.py`
