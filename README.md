# RNA-Foundation-FineTuning

Official repository for the Bachelor's Thesis: **"Design of fine-tuning pipelines in transcriptomic foundation models for the analysis of omics datasets"**.

This project explores the adaptation of deep learning architectures applied to transcriptomics, comparing the predictive performance and biological knowledge extraction of classical Machine Learning models against the transcriptomic foundation model **GeneRAIN**. Furthermore, it implements parameter-efficient fine-tuning using **LoRA** and utilizes Tabular Variational Autoencoders (**TVAE**) to mitigate data scarcity through the generation of synthetic samples.

## 🎯 Project Objectives
1. **Evaluate predictive performance:** Compare classical approaches (LASSO, Random Forest, MLP) on raw data against the latent projections (200-dimensional embeddings) generated by GeneRAIN, both in its frozen version (zero-shot) and after adaptation with LoRA.
2. **Biological interpretability:** Determine whether the foundation model's attention mechanisms favor the identification of genetic signatures with greater functional richness and biological specificity, quantified through the Mean Information Content (MIC) and Functional Abundance (FA).
3. **Data augmentation:** Evaluate the impact of incorporating synthetic transcriptomic data (generated with TVAE) to mitigate noise and sample scarcity in complex clinical cohorts, such as Parkinson's Disease (PPMI).

## ⚙️ Workflow (Pipeline)

The code is structured into a sequential experimentation flow where the output of one stage feeds into the next:

*   **1. Data Download:** The starting point consists of downloading the raw transcriptomic count files from public repositories such as **GTEx**, ARCHS4 and PPMI.
*   **2. Preprocessing and Gene Selection (`split.py`, `DEG.py`):** These scripts strictly take the raw count matrices as input. `split.py` performs data partitioning (80/10/10) and extracts highly variable genes (HVG). Meanwhile, `DEG.py` takes the raw counts and clinical labels to identify differentially expressed genes using PyDESeq2.
*   **3. Data Augmentation (`TVAE.py`):** Based on the previous training partition and the filtered gene lists, it synthesizes transcriptomic profiles using a Tabular Variational Autoencoder to balance and augment the available data.
*   **4. Classical Evaluation (`baseline.py`):** This script takes the combined data generated by `split.py`, `DEG.py`, and `TVAE.py`. It optimizes and evaluates the baseline models (LASSO, Random Forest, MLP) on the standardized data.
*   **5. Latent Extraction - Phase 1 (`phase1.py`):** Similar to the baseline, this script takes the previously processed data from `split.py`, `DEG.py`, and `TVAE.py`, along with the original pre-trained weights of GeneRAIN. It applies the *Binning-By-Gene* normalization, extracts the static *embeddings* from the frozen model (zero-shot), and trains an MLP classifier.
*   **6. Fine-Tuning - Phase 2 (`phase2.py`):** To run, this script requires the data, features, and base model generated in Phase 1. It performs the fine-tuning of the architecture by injecting LoRA adapters into the *Transformer*'s attention matrices.
*   **7. Biological Interpretability (`run_enrichment.py`, `sweep_topk.py`, `plot_sweep.py`):** 
    *   `run_enrichment.py` takes the gene rankings (LASSO coefficients, RF importance, and GeneRAIN's attention matrices) along with the *background* file (gene universe of the problem). It connects to the g:Profiler API and returns the static MIC and FA metrics.
    *   `sweep_topk.py` takes the results from the phases and ranking metrics to automate the calculation by performing a dynamic sweep of the Top-K variable, generating a global CSV file.
    *   `plot_sweep.py` takes the previous global CSV to generate the final multiple-sensitivity line plots in `.png` format.

## 🗂️ Case Studies

The project has been empirically validated across three scenarios with varying levels of predictive difficulty:
1.  **Trivial Difficulty (Positive Control):** Tissue differentiation between brain frontal cortex and blood (GTEx/ARCHS4 data).
2.  **Intermediate Difficulty:** Morphological classification of tissues from the same organ (sigmoid colon vs. transverse colon).
3.  **High Difficulty:** Detection of idiopathic Parkinson's Disease in peripheral blood samples, using the Parkinson's Progression Markers Initiative (PPMI) clinical cohort.

## 🛠️ Requirements and Execution Environment

To ensure computational reproducibility, running this project in a Conda environment using a GPU is recommended. Main dependencies include:
*   `python` 3.13.12
*   `pytorch` and the CUDA ecosystem for accelerated training.
*   `calm-data-generator` for synthetic data generation.
*   `pydeseq2`, `optuna`, `scanpy`, `anndata`, `umap-learn`, `gprofiler-official`.

**⚠️ Important note on GeneRAIN:** 
To run the Deep Learning architecture (Phases 1 and 2), **it is strictly necessary to have previously followed the steps from the official GeneRAIN repository**. You must clone their repository, install its native dependencies, download the pre-trained model weights, and correctly locate the corresponding `.json` configuration files, as detailed in their official documentation.

## 🚀 Usage (Execution)

Given the strict data dependency flow, scripts must be executed in order:

```bash
# 1. Generate partitions and extract genes (requires raw counts)
python src/split.py
python src/DEG.py

# 2. Generate synthetic data
python src/TVAE.py

# 3. Evaluate classical baseline and extract features with GeneRAIN
python src/baseline.py
python src/phase1.py

# 4. Execute fine-tuning (requires phase1.py data)
python src/phase2.py 

# 5. Extract and plot latent biological knowledge
python src/run_enrichment.py
python src/sweep_topk.py
python src/plot_sweep.py
```

# RNA-Foundation-FineTuning

Repositorio oficial del Trabajo de Fin de Grado: **"Diseño de pipelines de ajuste fino en modelos fundacionales de RNA para el estudio de datasets de muestras ómicas"**.

Este proyecto explora la adaptación de arquitecturas de aprendizaje profundo (Deep Learning) aplicadas a la transcriptómica, comparando el rendimiento predictivo y la extracción de conocimiento biológico de modelos clásicos de Machine Learning frente al modelo fundacional transcriptómico **GeneRAIN**. Además, implementa un ajuste fino paramétricamente eficiente mediante **LoRA** y utiliza Autoencoders Variacionales Tabulares (**TVAE**) para mitigar la escasez de datos mediante la generación de muestras sintéticas.

## 🎯 Objetivos del Proyecto
1. **Evaluar el rendimiento predictivo:** Comparar enfoques clásicos (LASSO, Random Forest, MLP) sobre datos en crudo frente a las proyecciones latentes (embeddings de 200 dimensiones) generadas por GeneRAIN, tanto en su versión congelada (zero-shot) como tras su adaptación con LoRA.
2. **Interpretabilidad biológica:** Determinar si los mecanismos de atención del modelo fundacional favorecen la identificación de firmas genéticas con mayor riqueza funcional y especificidad biológica, cuantificado a través del Contenido de Información Promediado (MIC) y la Abundancia Funcional (FA).
3. **Aumento de datos:** Evaluar el impacto de incorporar datos transcriptómicos sintéticos (generados con TVAE) para mitigar el ruido y la escasez de muestras en cohortes clínicas complejas, como la Enfermedad de Parkinson (PPMI).

## ⚙️ Flujo de Trabajo (Pipeline)

El código está estructurado en un flujo de experimentación secuencial donde la salida de una etapa alimenta a la siguiente:

*   **1. Descarga de datos:** El punto de partida consiste en descargar los ficheros de conteos transcriptómicos en crudo desde repositorios públicos como **GTEx** y ARCHS4.
*   **2. Preprocesamiento y Selección de Genes (`split.py`, `DEG.py`):** Estos scripts reciben como entrada estricta las matrices de conteos en crudo. `split.py` realiza el particionamiento de los datos (80/10/10) y extrae los genes de alta varianza (HVG). Por su parte, a `DEG.py` se le meten los conteos en crudo y las etiquetas clínicas para identificar los genes diferencialmente expresados mediante PyDESeq2.
*   **3. Aumento de Datos (`TVAE.py`):** A partir de la partición de entrenamiento anterior y las listas de genes filtrados, sintetiza perfiles transcriptómicos utilizando un Autoencoder Variacional Tabular para balancear y aumentar los datos disponibles.
*   **4. Evaluación Clásica (`baseline.py`):** A este script se le meten conjuntamente los datos generados por `split.py`, `DEG.py` y `TVAE.py`. Optimiza y evalúa los modelos de referencia (LASSO, Random Forest, MLP) sobre los datos estandarizados.
*   **5. Extracción Latente - Fase 1 (`phase1.py`):** Al igual que a la baseline, a este script se le meten los datos procesados previamente por `split.py`, `DEG.py` y `TVAE.py`, junto con los pesos preentrenados originales de GeneRAIN. Aplica la normalización *Binning-By-Gene*, extrae los *embeddings* estáticos del modelo congelado (zero-shot) y entrena un clasificador MLP.
*   **6. Fine-Tuning - Fase 2 (`phase2.py`):** Para poder ejecutarse, a este script hay que meterle los datos, características y el modelo base generado en la Fase 1. Realiza el ajuste fino de la arquitectura inyectando adaptadores LoRA en las matrices de atención del *Transformer*.
*   **7. Interpretabilidad Biológica (`run_enrichment.py`, `sweep_topk.py`, `plot_sweep.py`):** 
    *   A `run_enrichment.py` se le meten los rankings de genes (coeficientes de LASSO, importancia de RF y las matrices de atención de GeneRAIN) junto con el archivo de *background* (universo de genes del problema). Conecta con la API de g:Profiler y devuelve las métricas estáticas de MIC y FA.
    *   A `sweep_topk.py` se le introducen los resultados de las fases y métricas de ranking para automatizar el cálculo realizando un barrido dinámico de la variable Top-K, generando un archivo CSV global.
    *   A `plot_sweep.py` se le mete el CSV global anterior para generar los gráficos finales de líneas de sensibilidad múltiple en formato `.png`.

## 🗂️ Casos de Estudio

El proyecto se ha validado empíricamente a través de tres escenarios con niveles de dificultad predictiva variables:
1.  **Dificultad Trivial (Control Positivo):** Diferenciación tisular entre córtex frontal del cerebro y sangre (datos de GTEx).
2.  **Dificultad Intermedia:** Clasificación morfológica de tejidos del mismo órgano: colon sigmoide vs. colon transverso (datos de GTEx).
3.  **Alta Dificultad:** Detección de la Enfermedad de Parkinson idiopática en muestras de sangre periférica, empleando la cohorte clínica de Parkinson's Progression Markers Initiative (PPMI).
